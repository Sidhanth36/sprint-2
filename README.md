# sprint-2
NHS Patient-Centric Hospital System – Sprint 2

Sprint 2 focused on extending the work completed in Sprint 1 by improving the system structure and developing additional functionality for the NHS Patient-Centric Hospital System. The aim of this sprint was to build on the project foundation and move closer to a more functional prototype.

During this sprint, the team continued working on the selected user stories and further developed key system features. Particular attention was given to authentication and login functionality, appointment booking, and doctor availability. These features were important because they directly support the patient experience and reflect the main goals of the hospital system described in the project brief.

The sprint also involved refining the product backlog, preparing a clearer sprint backlog, and tracking the work completed during this phase of development. In addition to technical work, the team reviewed progress, identified issues, and discussed improvements through sprint review and retrospective activities.

The project continued to be developed using C# and the .NET Framework. GitHub remained an important tool during Sprint 2, as it was used to manage updated files, track changes, and record the progress made by the team. The repository showed evidence of feature improvements, system updates, and collaborative work during the sprint.

Sprint 2 was an important stage in the project because it transformed the early work from Sprint 1 into a more developed and usable system prototype.
